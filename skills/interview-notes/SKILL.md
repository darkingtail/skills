# Interview Notes

将面试复习的知识点记录到笔记文件中。

## 触发条件

当用户满足以下任一条件时使用：

1. **明确要求记录** — "记录一下"、"记下来"、"保存这个知识点"
2. **复习完一个知识点** — "这个过了"、"下一个"、"复习完了"
3. **主动要求保存** — "保存笔记"、"更新笔记"

## 执行步骤

1. 读取当前笔记文件 `{Obsidian workspace}/interview-notes.md`（如果不存在则创建）
2. 将本次复习的知识点追加到对应分类下
3. 每条记录包含：
   - 知识点标题
   - 核心要点（3-5 条）
   - 常见追问
   - 掌握程度：`[ ] 不熟` / `[~] 一般` / `[x] 掌握`

## 笔记文件格式

```markdown
# 前端面试复习笔记

## P0 必考

### JavaScript 核心

#### 事件循环
- **要点**：同步 → 微任务（全部清空）→ 可能渲染 → 一个宏任务 → 循环
- **追问**：async/await 与事件循环关系、Vue nextTick 实现
- **掌握**：[x] 掌握

#### 闭包
- **要点**：...
- **追问**：...
- **掌握**：[ ] 不熟

### Vue 原理
...

## P1 高频
...

## P2 中频
...
```

## 分类结构

```
P0 必考
├── JavaScript 核心（闭包、原型链、this、事件循环、Promise）
├── Vue 2/3 原理（响应式、虚拟 DOM、Diff、nextTick）
└── 浏览器与网络（HTTP 缓存、CORS、渲染流程）

P1 高频
├── CSS 布局（Flex/Grid、BFC、居中、层叠上下文）
├── 构建工具（Webpack/Rspack 原理、tree shaking）
└── 手写题（Promise.all、防抖节流、深拷贝）

P2 中频
├── TypeScript（泛型、type vs interface）
├── 微前端（JS 沙箱、CSS 隔离）
└── 性能优化（首屏、懒加载、虚拟列表）
```

## 注意事项

1. **只追加不覆盖** — 每次只在对应分类下追加新知识点
2. **保持格式统一** — 严格遵循上述 markdown 格式
3. **标记掌握程度** — 根据用户回答质量判断
4. **记录用户的薄弱点** — 方便下次针对性复习
