# Today I Learned (每日一练)

## 存储规则

**路径**: `{Daily TIL}/{年份}/{月-日}.md`（路径从 CLAUDE.md `# Paths` 解析）

**目录结构示例**:
```
{Daily TIL}/
├── 2025/
│   ├── 12-01.md
│   └── 12-15.md
└── 2026/
    ├── 01-05.md
    ├── 02-11.md
    └── 02-12.md
```

## 执行步骤

1. **获取当前日期**: 用当前日期生成路径
   - 年份目录: `2026`
   - 文件名: `02-13.md`

3. **确保目录存在**: 如果年份目录不存在则创建

4. **写入知识点**:
   - 如果当日文件**不存在**: 创建新文件，写入标题 + 第一条知识点
   - 如果当日文件**已存在**: 读取现有内容，追加新知识点到末尾

## 文件格式

```markdown
# {年}-{月}-{日} 知识点

## 1. {知识点标题}

{知识点内容}

---

## 2. {知识点标题}

{知识点内容}

---
```

## 写入规范

- 每条知识点用 `## {序号}. {标题}` 作为标题
- 追加时自动递增序号（读取已有内容计算下一个序号）
- 知识点之间用 `---` 分隔
- 内容简洁明了，包含关键命令或代码示例
- 用中文记录（除非知识点本身是英文技术术语）
- 不要重复记录当日已有的相同知识点
